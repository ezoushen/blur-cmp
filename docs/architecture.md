The two platforms use **fundamentally different strategies**:
- **Android**: CPU capture вҶ’ GPU blur вҶ’ CPU readback вҶ’ View draw
- **iOS**: Zero-copy GPU compositor capture via private `CABackdropLayer`

## 2. Android Pipeline

### 2.0 RenderEffect Fast Path (API 31+, uniform blur)

When conditions are met (API 31+, no gradient, Normal blend mode tint),
BlurOverlayHost bypasses the entire View-based pipeline:

```
  background()                    вҶҗ unblurred (always rendered at Box root)
  Box(graphicsLayer {
    renderEffect = BlurEffect     вҶҗ GPU blur on RenderThread
    alpha = state.alpha           вҶҗ GPU alpha modulation
  }) {
    background()                  вҶҗ re-rendered inside blurred layer
  }
  Box(tint overlay)               вҶҗ crisp tint, outside blur layer
  content()                       вҶҗ sharp, on top

  CPU-GPU crossings: 0
  CPU involvement: zero pixel access
```

Falls through to Kawase pipeline (Section 2.1+) when:
- API < 31
- Gradient/variable blur configured
- Non-Normal blend mode tint (requires pre-blur bitmap access)

  в”Ӯ     в—ү   в—ү                 в”Ӯ
  в”Ӯ   в—Ӣ   в—Ӣ   в—Ӣ               в”Ӯ

  Iteration count from radius (logarithmic):
    floatIter = log2(radius / BASE_SIGMA)
    iterations = floor(floatIter)  вҶҗ clamped [1, 8]
